How to install

Before you start

You'll need three things ready before you add the widget to your site. Have them open in tabs as you work through the steps.

  • Access to your RPR account at narrpr.com with at least one published market report.
  • A way to receive leads — an email inbox, a phone notification, a spreadsheet, or a CRM. You'll choose one in Step 2.
  • The ability to add custom HTML to a page on your website — either your own login, or someone who can paste a snippet for you.
Not sure you can add code? If your site builder has a Custom HTML, Code, or Embed block, you're all set. If you're not sure, your web designer or site manager can add the snippet in a few minutes.
01

Get your report URLs from RPR

Every area you offer in the widget needs a link to a live RPR market report.

  1. Log into narrpr.com.
  2. Click Reports in the top navigation.
  3. Run a Market Activity Report for the ZIP code or neighborhood you want to feature — or open one you've already published.
  4. Click Share and copy the public PDF link. It will look like this:
https://www.narrpr.com/reports-v2/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx/pdf

Save the link somewhere you can paste from later. Repeat for each area you plan to offer.

Start small. Three to five areas is plenty. A dropdown with twenty options is harder to navigate than one with five. You can always add more later.
02

Decide how you want to receive leads

When someone submits the form, their info is sent to a webhook URL — a web address that hands the data off to your phone, inbox, spreadsheet, or CRM. The generator's Lead Delivery step has a “How do you want to receive leads?” dropdown that walks you through each option below and gives you the exact URL to paste. Pick the one that matches how you work:

If you want…Use this
An email when someone submitsToolkit.app — free, ~2-minute setup, no account required. Leads arrive in your inbox.
A push notification on your phonentfy.sh or SimplePush (both free), or Pushover ($5 one-time)
Leads in a spreadsheetGoogle Sheets — free. The generator gives you a ready-made template with the receiver script already attached; no code to paste.
A message in your team chatSlack or Discord — both free
Leads in GoHighLevelGHL's native Inbound Webhook trigger (Automation → Workflow → Inbound Webhook)
Leads in another CRMBridge with Make ($9/mo) or Zapier (paid plan), or paste your CRM's own inbound webhook URL under Custom
To connect to almost anythingAny Custom HTTPS webhook — the widget POSTs the lead as JSON

Whichever tool you choose, the goal is the same: copy the webhook URL it gives you. Keep it handy — you'll paste it in the next step.

The webhook URL must start with https://. The widget refuses any non-HTTPS URL and disables delivery, so leads wouldn't be captured.
Heads up on simple form services. The widget sends each lead as a JSON POST. Tools that expect a classic HTML form post with their own hidden access-key field (for example Web3Forms) won't read it correctly — use one of the methods in the dropdown instead.
03

Build your embed code in the generator

You build your embed code in the generator at the top of this page. It has a configuration panel on the left, a live preview on the right, and your finished embed code at the bottom. Every change updates everything instantly.

Fill in, in order:

  1. Market reports — type an Area label (e.g. Scottsdale, AZ 85254) and paste the matching report URL from Step 1. Click Add area for each one.
  2. Lead delivery — pick a method from the dropdown and paste the webhook URL from Step 2. The “no webhook URL” warning disappears once you do.
  3. Branding — agent name, brokerage, logo URL (must start with https://), and a brand color.
  4. Copy — optional. Adjust the headline, subheadline, and button text if you want to match your voice.
  5. Display — choose Inline (form on a specific page), Floating (a button on every page), or Modal (popup triggered by your own button).
Tip — click “Send test” before you finish. Once you've pasted a webhook URL, a Send test button appears under it. It fires a sample submission through your webhook so you know the connection works before the widget goes live.
Your work is saved automatically. The generator remembers your settings in your browser, so they're still there when you come back. To move your setup to another computer — or keep a backup — click Copy generator link and save that link; opening it reloads your exact configuration. Reset clears everything and starts fresh.

Copy your embed code

Scroll to the Embed code box and click Copy. The code looks roughly like this:

<!-- RPR Market Reports Widget --> <script src="https://pub-6607a59d1d3b4ed18490937c995526d1.r2.dev/rpr-reports-embed.js" data-reports='[{"label":"Scottsdale, AZ 85254","url":"https://www.narrpr.com/reports-v2/…/pdf"}]' data-webhook="https://ntfy.sh/your-topic-name" data-agent-name="Your Name" data-brokerage="Your Brokerage" data-form-mode="full" data-display-mode="inline"></script>

Your actual code will include your own areas, branding, and webhook. The shape is the same: a single <script> tag.

04

Add the code to your website

The embed code is a single <script> tag. It works anywhere you can add custom HTML or JavaScript — and only there. No plugin, no special integration: you're just pasting the snippet into a spot your platform already provides.

Put it on one page — Inline or Modal mode

In your page editor, find a block called Custom HTML, Code, or Embed — not a plain text block — and paste the embed code into it. Save and publish.

Put it on every page — Floating mode

Find your site's global header / footer code or custom code area (often under Site or Theme Settings → Custom Code). Paste the embed code there once and it appears on every page.

Not every platform allows custom code. Some hosted website platforms — Luxury Presence, for example — restrict custom JavaScript, so you may need to ask their support to add it, or it may not be supported at all. Squarespace's free plan and social-media profiles don't allow it either. If you're unsure, ask your platform's support one question: “Can I add a custom <script> tag to my site?” — that tells you everything.
Can't find the spot? If your platform accepts custom HTML or JavaScript somewhere, the widget works there. If it doesn't, it can't run there. Either way, your web designer can place a single script tag in a few minutes — just send them the embed code.

Three display modes work on every platform

Whichever platform you use, the embed code supports three display modes (set in the generator's Display step):

  • Inline — the form appears on one specific page where you paste the code. Best for a dedicated landing page.
  • Floating — a button stays fixed to a corner on every page. Add the code once in the footer / global scripts of your site. Best for site-wide capture.
  • Modal — the form opens as a popup when someone clicks a button you already have. Install the code site-wide, then in the generator set Display → Modal and enter a Trigger selector — a CSS selector that matches your button (for example #open-reports or .report-button). Clicking that button opens the form.
05

Send a test lead from your live page

One last check — a real submission from the real page, to make sure your platform, the widget, and your webhook are all talking to each other.

  1. Open the public page where you installed the widget (not the preview — the live URL).
  2. Fill out the form with a test name, your own email, and pick any area.
  3. Click submit. You should see the confirmation screen with a button that opens the RPR report.
  4. Check your delivery destination:
    • ntfy / SimplePush / Pushover — a notification appears on your phone within seconds.
    • Toolkit.app — an email should arrive within a minute (check spam the first time).
    • Slack / Discord — a message appears in your channel.
    • Google Sheets — a new row appears in your sheet.
    • GoHighLevel — a new contact / workflow run appears.
    • Make / Zapier — check the scenario or task history; you should see one run.
If the test lead arrived, you're done. Your widget is live and capturing leads. Skip ahead to the FAQ only if something's off.
FAQ & Troubleshooting

Questions we hear most often.

Grouped by topic. Use the search below if you know what you're looking for, or click through each group.

Common setup questions

10 questions
Do I need to know how to code?

No. The generator at the top of this page builds the embed code for you. You fill in a form, copy the result, and paste it into your website. If you can add a YouTube video to a page, you can add this widget.

Which website platforms does this work on?

Any platform that lets you add custom HTML or JavaScript — and only those. That covers most open and self-hosted site builders: WordPress (Avada, Gutenberg, Elementor, Divi, etc.), Squarespace paid plans, Wix, Webflow, Showit, and the like.

Some closed, hosted platforms restrict custom JavaScript — Luxury Presence is one example — so there you may need their support to add it, or it may not be supported at all. The quickest way to know: ask your platform's support whether you can add a custom <script> tag. If you can, the widget works; if you can't, it can't run there.

Do I need a Zapier account?

No — and Zapier actually requires a paid plan for its webhook trigger. It's just one of many options, and several free ones work great:

  • Just want an email when someone submits — Toolkit.app (free, ~2-minute setup, no account)
  • A notification on your phone — ntfy.sh or SimplePush (free), or Pushover ($5 one-time)
  • Leads in a spreadsheet — Google Sheets (free; the generator gives you a ready-made template)
  • A message in team chat — Slack or Discord (free)
  • Using GoHighLevel — its native inbound webhook, no third-party tool

The generator's “How do you want to receive leads?” dropdown walks you through each one.

Does it work with my CRM?

GoHighLevel connects natively — it has an inbound webhook trigger you paste straight into the generator. For other CRMs (Follow Up Boss, HubSpot, Salesforce, kvCORE, and the like), use Make or Zapier as a bridge, or paste the CRM's own inbound webhook URL into the Custom option if it has one. The widget sends each lead as JSON, so the receiving end needs to accept a JSON webhook.

Can I offer more than one area?

Yes. The generator lets you add as many area and report pairs as you want. Each one appears as an option in the dropdown. There's no hard limit, but a list of 20 areas is harder for a visitor to navigate than a list of 5. Focus on your primary markets.

Can I use this on more than one website?

Yes. Generate a separate embed code for each website — each can have different branding, different areas, and a different webhook URL. All of them run off the same hosted script, so when the script is updated, every site gets the improvement automatically with no action on your part.

Can two agents at the same brokerage each have their own version?

Yes. Each agent generates their own embed code with their own name, branding, and webhook. They are completely independent.

What does the visitor see after they submit?

The form transitions to a confirmation screen showing a card with the report for the area they selected. The card displays the area name, a brief heading, and a View Report button that opens the RPR PDF report in a new tab.

Is the visitor's information stored anywhere?

No. The widget itself doesn't store any data. The only place lead data goes is your webhook endpoint — your notification app, spreadsheet, CRM, or wherever you configured. If you haven't set up a webhook, no data is saved anywhere.

Can I add this to my Facebook or Instagram page?

No. Social media platforms don't allow custom JavaScript. The widget works on websites you control — your own domain or a website builder that allows custom code.

The form isn't showing up on my page

4 checks
Check that you pasted the full code

The embed code starts with <script and ends with ></script>. Both parts must be present. If anything got cut off during copy-paste, go back to the generator and copy it again using the Copy button.

Check that you used a code/embed block, not a text block

On almost every platform, code pasted into a regular text or paragraph block displays as plain text rather than executing. Make sure you used a Custom HTML, Code, or Embed block — whatever your platform calls it — not a plain text or paragraph block.

Check for platform or hosting restrictions

Some WordPress hosting configurations — and Squarespace's free plan — block custom JavaScript. If the form shows up in the generator preview but not on your live page, your platform or host may be blocking scripts. Contact your hosting support to confirm.

Security plugins like Wordfence or Sucuri occasionally flag third-party scripts. If you use one, check its dashboard for blocked requests.

Try a hard refresh

After adding the code, press Ctrl + Shift + R on Windows or ⌘ + Shift + R on Mac to force your browser to reload the page without cache. If you use a caching plugin (W3 Total Cache, WP Rocket, etc.), clear its cache from the WordPress admin as well.

I'm not receiving leads

5 checks
Confirm your webhook URL is set

Go back to the generator's Lead Delivery step and check the Webhook URL field. If it's empty, a warning appears (“No webhook URL — leads won't be captured”) — choose a delivery method, paste the URL, and copy the code again.

Confirm your delivery tool is turned on

If you're routing leads through Zapier, Make, a HubSpot workflow, or a Pabbly workflow, make sure the Zap / scenario / workflow is switched On. One that's off receives nothing — this on/off switch is the single most-missed step.

Send a test from the generator

In the generator, once you've pasted a webhook URL, click the Send test button beneath it. Then check your delivery tool.

  • If your tool shows the lead came in but you're not getting notified, the issue is in your tool's action step — check its notification / email configuration.
  • If your tool shows nothing came in, the issue is in the webhook connection itself — the URL is wrong or the automation is off.
Check that your webhook URL starts with https://

The widget blocks webhooks that don't use HTTPS. A URL starting with http:// is rejected and delivery is disabled, so no lead data is sent. Copy the URL fresh from your delivery tool if you're unsure.

Check your spam folder

If your delivery tool (Toolkit.app, a Zapier email step, etc.) sends you an email and you're not seeing it, check spam or junk mail. Add the sending address to your contacts to prevent future filtering.

The submit button isn't working

3 checks
Fill in all required fields

In the default full form, first name, last name, email, and an area selection are required (phone is optional). If any required field is missing or invalid, the form highlights it and won't submit. If you switched the form to minimal mode in the generator, only email and an area are required.

Check the GDPR checkbox if it is visible

If you enabled the GDPR consent checkbox in the generator (Settings → Privacy), the form cannot be submitted until that box is checked.

Try a different browser or incognito window

Occasionally a browser extension — particularly ad blockers or tracker blockers like uBlock Origin or Privacy Badger — will interfere with form submission. Try submitting in a private or incognito window, or a different browser, to rule this out.

The colors or fonts don't look right

3 checks
Allow a few seconds after loading

Google Fonts load after the page does — there may be a brief moment where a fallback font appears before the custom font takes over. This is normal and only affects the first visit; subsequent visits are instant because the font is cached.

Check your hex color code

Color values must be a valid 3 or 6 digit hex code starting with # — for example #1a1a2e or #fff. If you typed a color name like navy or a value without the #, the widget falls back to the default color.

Clear your browser cache

If you updated your embed code and your changes aren't showing, your browser may be serving a cached version. Hard-refresh with Ctrl + Shift + R (Windows) or ⌘ + Shift + R (Mac), and clear any WordPress caching plugin too.

The report link isn't opening

3 checks
Check the report URL you pasted into the generator

It should be the public PDF link from RPR's Share button, like this:

https://www.narrpr.com/reports-v2/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx/pdf

The widget accepts any link that starts with https://. The generator shows a “doesn't look like an RPR report URL” warning if your link isn't from narrpr.com — it won't block you, but it's a sign you may have copied the wrong link. Double-check you used Share → PDF.

Check that the report is still published

RPR reports can expire or be unpublished. Log into narrpr.com and confirm the report is still accessible. If it's expired, run a new report and update the URL in your embed code.

Check whether the visitor is logged in to RPR

Some RPR report links require the viewer to log in to an RPR account to access them. If your visitors are not REALTORS® with RPR access, make sure you're using a publicly shareable link. When you click Share on a report in RPR, confirm it generates a public link rather than a member-only one.

Updates & multiple pages

3 questions
How do I update my areas or branding later?

Come back to the generator, make your changes, and copy the new embed code. Replace the old embed code in your code/embed block (or in the global footer field, if you used Floating mode) with the new one. The old and new codes are both valid at the same time — there's no deactivation step.

Tip: the generator remembers your settings in your browser automatically, so your work is usually still there. For a portable backup or to edit from another computer, click Copy generator link and save that link — opening it reloads your exact configuration.

Can I add the widget to multiple pages on my site?

Yes. Paste the same embed code into a code/embed block on as many pages as you want. Each page will show an independent copy of the form. If you're using Floating mode, you only need to add it once — it appears on every page automatically.

I've worked through everything above and it's still not working

Hand the problem to your web designer or IT contact with the following information:

  1. The embed code from the generator (copy it fresh)
  2. The name of your website platform (WordPress + Avada, Squarespace, Wix, etc.)
  3. A description of what you see — “the form doesn't appear at all” or “the form appears but leads aren't arriving”
  4. A link to the page where you added the code

Most web professionals can resolve embed issues in under 30 minutes with this information.

Still stuck after all that?
